# AGENTS.md

## 目标

本仓库要求所有有实质影响的修改都留下可追踪的进度记录，并由用户手动决定何时推送到远程仓库。

## 默认工作流

1. 任何有实质影响的代码、脚本、配置、产品文档修改，都必须同步更新 `docs/product/知库_开发变更日志.md`
2. `docs/product/知库_开发进度同步.md` 保留为高层阶段性进度文档，不承担逐次改动流水账职责
3. 在交付本轮修改前，必须补齐以下内容：
   - 改了什么
   - 改动文件
   - 如何验证
   - 当前遗留
   - 下一步建议
4. 除非用户明确要求，否则不要执行 `git push`、不要自动创建远程分支、不要改写远程历史
5. 当用户准备手动推送前，优先提醒先运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev\pre_push_check.ps1
```

## 进度日志规则

- `docs/product/知库_开发变更日志.md` 使用“最新记录在前”的方式维护
- 每次改动批次至少要写明：日期、任务、状态、改动文件、验证、遗留、下一步
- 如果本轮只是只读分析且没有落地改动，可以不新增日志条目

## 推送规则

- 默认流程是：实现 -> 验证 -> 更新进度文档 -> 用户 review -> 用户手动 `commit/push`
- 若用户只要求“准备好推送”，则停在本地可提交状态，不替用户自动推远端
