# 知库 Common Ground

## 1. 文档目的

本文档用于记录当前 `知库` 项目的关键共识，避免后续开发出现方向漂移。

参考思路来自 `claude-skills` 的 `Common Ground`，但这里是适配 `知库` 的项目版本。

---

## 2. ESTABLISHED（已确认共识）

### 产品定位

`知库` 是一个个人内容采集、知识整理、知识问答产品。

### 当前主闭环

当前主闭环固定为：

`导入链接/文件 -> 解析正文 -> 整理笔记 -> 入库 -> 问答 -> 沉淀知识卡片`

### 当前优先数据源

- B站视频
- 网页 / 文章 URL

### 当前首要目标

不是上线，不是商业化，而是：

**先做出一个你自己能稳定验证、并且普通用户也能看懂的产品原型。**

### 当前技术形态

- 前端：`React + Vite`
- API：`FastAPI`
- 存储：`SQLite + FTS`
- 问答：本地知识库检索 + OpenAI 兼容模型

### 当前产品语言要求

- 中文
- 极简
- 产品化
- 不要后台感 / 调试感

---

## 3. WORKING（当前工作假设）

### RAG 路线

当前认为最适合 `知库` 的路线是：

- 先做 `ragent-lite`
- 先提升问题改写、多路召回、上下文参与
- 后续再补真正向量检索与重排增强

### 模型路线

当前优先使用：

- OpenAI 兼容国产模型接口

原因：

- 接入快
- MVP 便于验证
- 成本更低

### 采集策略

视频正文优先级：

1. 字幕
2. 页面文本
3. 音频转写

网页正文优先级：

1. 页面正文
2. Meta 信息
3. 用户手动补充

---

## 4. OPEN（仍需持续验证）

### 解析成功率

- B站复杂内容、分 P 视频、受限内容仍需持续验证
- 网页正文抽取对复杂站点的稳定性仍需继续验证

### 问答质量

- 当前混合检索已比纯关键词好，但仍未达到成熟 RAG 水平
- 还需要继续补：
  - 重排
  - 回答质量分级
  - 失败回退

### 普通用户体验

- 目前仍然偏“原型产品”
- 还需要继续降低认知成本

---

## 5. 开发前检查清单

后续每次开发前，默认先检查：

1. 本次任务属于哪一层
2. 是否服务主闭环
3. 是否已有现成能力可复用
4. 是否会增加普通用户理解成本
5. 应该用哪种专家视角来实现

---

## 6. 开发后同步清单

后续每次开发后，默认同步：

1. 改了什么
2. 为什么这样改
3. 验证了什么
4. 当前还有什么问题
5. 下一步是什么

---

## 7. 当前一句话共识

`知库` 现在最重要的不是继续扩展边界，而是把：

**B站 / 网页内容采集 + 知识整理 + RAG 问答**

这条主链路做深、做稳、做聪明。
