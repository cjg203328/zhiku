## 本次完成

### 1. 真实 B站链接验证

本次使用真实链接验证：

- `https://www.bilibili.com/video/BV1BMPszZER2/`

验证结果：

- 当前公开视频接口可以拿到基础元数据和音频流。
- 当前公开视频接口没有直接返回字幕正文。
- `x/player/wbi/v2` 返回 `need_login_subtitle = true`。
- 因此这条内容在未配置 Cookie、未配置 ASR 时，只能完成“基础建档”，不能视为真正的内容转化成功。

### 2. B站转化状态治理

后端新增了显式采集状态：

- `ready`
- `ready_estimated`
- `needs_cookie`
- `needs_asr`
- `asr_failed`
- `limited`

并新增以下关键元数据：

- `capture_status`
- `capture_quality`
- `capture_summary`
- `capture_recommended_action`
- `capture_blocked_reason`
- `subtitle_login_required`
- `subtitle_preview_toast`
- `audio_available`
- `asr_configured`

这样系统可以区分：

- 已获得可回溯正文
- 已恢复正文但时间戳为估算
- 当前受 B站登录态限制
- 当前需要音频转写
- 当前转写链路失败
- 当前仅完成基础建档

### 3. ASR 主链路增强

本次把 ASR 从“只收一整段文本”升级为“优先尝试结构化分段转写”：

- 优先请求 `verbose_json`
- 尝试读取 `segments`
- 如果 ASR 返回真实分段时间戳，则直接生成时间化转写片段
- 如果 ASR 只返回整段文本，则继续使用现有估算时间戳分段
- 如果结构化返回失败，则自动回退到普通 `json`

这意味着后续接通 OpenAI-compatible ASR 后，可以形成更接近目标形态的链路：

- `音频 -> 带时间戳转写 -> 检索片段 -> 时间点跳转`

### 4. 问答保护逻辑

问答层新增“来源待补全”保护：

- 如果 scoped 问答命中的内容状态为 `needs_cookie / needs_asr / asr_failed / limited`
- 系统不再把它当成完整知识笔记来回答
- 而是明确告诉用户：当前内容尚未完成转化，下一步该补 Cookie 还是补 ASR

这样可以显著减少“表面有答案、实际依据很弱”的误导性输出。

### 5. 前端体验同步

导入页、详情页、知识库列表、问答页已同步这些状态：

- 导入页会把“高质量结果”和“待补 Cookie / 待补 ASR”分开
- 详情页会显示“转化状态”和“建议下一步”
- 知识库列表会把“仅基础建档”和“可继续验证”区分开
- 问答页在命中半成品内容时会提示用户去补全采集能力
