## 2026-03-24 第二十七轮：浏览器预览直连本地 API 修复
- 状态：已完成
- 任务类型：启动链路 / 预览环境 / 缺陷修复

### 本次已完成
- 修复 `apps/desktop` 在浏览器本地预览模式下默认 API 地址为空的问题：当页面运行在 `127.0.0.1` 或 `localhost` 时，前端会自动回落到 `http://127.0.0.1:38765`
- 放宽 API 的本地开发跨域规则，允许任意本机端口（如 `4173`、`1420`）访问，避免“API 实际已启动，但页面仍显示未连接”
- 重新构建桌面前端预览包，并重启本地预览服务与 API 服务，确认 `4173 -> 38765` 链路恢复

### 改动文件
- `apps/desktop/src/lib/api.ts`
- `services/api/src/zhiku_api/main.py`

### 如何验证
- `npm.cmd --prefix apps/desktop run build`
- `python -m py_compile services/api/src/zhiku_api/main.py`
- 打开 `http://127.0.0.1:4173`，确认首页“服务”状态不再显示“未连接”
- 访问 `http://127.0.0.1:38765/api/v1/health`，确认返回 `{"status":"ok","service":"zhiku-api","version":"0.1.0"}`
- 用带 `Origin: http://127.0.0.1:4173` 的请求访问 API，确认响应已包含允许本地预览跨域的头

### 当前遗留
- 目前浏览器本地预览默认回落到固定端口 `38765`，如果后续 API 改端口，需要同步调整或继续通过 `VITE_API_BASE_URL` 显式覆盖

### 下一步建议
- 如果后续会长期使用浏览器预览模式，可以补一个统一的 `start_desktop_preview.ps1` 脚本，把构建、API 启动和页面预览合并成单一入口
- 如果准备继续手动推送，先运行 `powershell -ExecutionPolicy Bypass -File .\scripts\dev\pre_push_check.ps1`
