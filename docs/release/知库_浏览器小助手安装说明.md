# 知库浏览器小助手安装说明

这份说明面向普通用户，目标是让 `知库` 自动接收你当前浏览器里的 B 站登录状态。

装好一次后，后面通常只要打开过一次 `bilibili.com`，知库就会自动连上，不需要你再手动复制 `Cookie`。

---

## 1. 小助手目录

本地目录：

```text
extensions/zhiku-bilibili-bridge
```

如果你当前就是在仓库里运行项目，也可以直接在设置页复制这一路径。

---

## 2. 安装步骤

如果你已经打开知库设置页，现在也可以直接点页面里的 `一键打开小助手` 按钮。

它会自动帮你打开：

- 浏览器扩展管理页
- 小助手本地目录
- `docs/release` 文档目录

这样你基本不用再自己找脚本或目录。

---

## 2.1 脚本方式

如果你不想自己找目录和地址，最简单的方法是先运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev\open_bilibili_bridge_helper.ps1
```

这个脚本同样会自动帮你打开：

- 浏览器扩展管理页
- 小助手本地目录
- `docs/release` 文档目录

你只需要在浏览器里点“加载已解压的扩展程序”，然后选中已经打开的那个文件夹。

### Chrome / Edge / 其他 Chromium 浏览器

1. 打开浏览器扩展管理页
2. 打开“开发者模式”
3. 选择“加载已解压的扩展程序”
4. 选中 `extensions/zhiku-bilibili-bridge`
5. 保持这个小助手处于启用状态

---

## 3. 第一次连接

1. 在知库设置页打开“自动连接浏览器（推荐）”
2. 浏览器访问一次 `https://www.bilibili.com`
3. 回到知库设置页，点“刷新连接状态”
4. 如果看到“已连上浏览器登录状态”，就说明已经成功

---

## 4. 日常使用

- 平时不用手动复制 `Cookie`
- 小助手会在你打开 B 站页面、登录状态变化、或定时检查时自动同步
- 登录状态只用于本地服务临时补全需要登录的视频，不长期写入仓库

---

## 5. 如果没有连上

按这个顺序排查：

1. 确认知库本地服务正在运行
2. 确认小助手仍然处于启用状态
3. 在浏览器里打开一次 B 站页面
4. 点一下浏览器工具栏里的知库小助手图标，手动触发一次同步
5. 回到知库设置页，再点“刷新连接状态”

---

## 6. 什么时候还需要手动方式

只有这些情况才建议回到手动方式：

- 当前浏览器不支持安装这个小助手
- 你暂时只想给一次性样本手动补登录状态
- 你在特殊环境里没有办法让浏览器小助手访问本地服务

这时再使用设置页里的“手动方式（高级）”即可。
